  1. http-kit http://http-kit.org

    这是用Java为Clojure写的HTTP server/client & Timer，支持WebSocket和HTTP长连，具有高性能，API简单且友好的特点。 它受到了不少社区的关注，已有developer参与贡献代码和文档，两度被顶上Hacker News首页，在 github 上也得到300多个star 我曾写过多个HTTP server／client
    • tiny-web-server https://github.com/shenfeng/tiny-web-server：C语言，pre-fork
    • dictionary http://dict.shenfeng.me/：C，epoll。2012年春节期间完成。有Android版本。
    • async-http-client https://github.com/shenfeng/async-http-client： 在netty基础上封装的异步client，支持SOCKS，HTTP代理，支持HTTPS
    • async-ring-adapter https://github.com/shenfeng/async-ring-adapter： Java & Clojure，在netty的基础上封装
    • nio-httpserver https://github.com/shenfeng/nio-httpserver：Java，学习Java NIO

  2. Rssminer http://rssminer.net

    RSS reader, 有点像Google Reader，但功能弱一些。 GR关闭后，受到一些关注。从这个项目学了到很多东西。 完整的单元测试，使我可以较轻松的维护它。为它，写了一些较基础的库
    • http-kit。需要一个异步server & client，所以有了它。
    • 模板引擎：mustache.clj：API简单且够用，高性能（数倍于另外的实现）。做前端几年，对什么样的好，有一些体会。
    • 数据库连接池：dbcp.clj。采用Threadlocal，可比市面上其它的都快很多，和http-kit的线程模型配合很好。
    • 中文分词：mmesg。正向最大匹配。准确性够用，速度够快，内存占用够小。
    一个为Rssminer feed定制的存储后端在计划中，打算用Go语言完成，预计能节省200%左右的存储，并能提供更好的性能。
